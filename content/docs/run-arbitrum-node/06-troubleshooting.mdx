### Step 2: Look for your scenario

Common troubleshooting scenarios and solutions are detailed below.

You can check logs by different log types: info, warn, and error.
#### Common Troubleshooting Scenarios

Below are common scenarios and their solutions organized by log type:

#### Info Log Messages
- `Unindex transactions` - Expected behavior when removing old transaction lookups
- `Head state missing, repairing` - Usually from ungraceful shutdown, should recover automatically
- `Resuming state snapshot generation` - Expected when starting with saved snapshot
- `Created TxLookup index` - Expected when generating transaction lookups

#### Warning Log Messages
- `reorg` - Expected when parent chain reorganizes blocks
- `Synchronisation failed, retrying` - Expected when connection issues occur, should self-correct
- `Rollback to unfinalized` - Expected during parent chain reorg

#### Error Log Messages
- `database contains incompatible genesis` - Check chain ID, may need to delete database
- `Error looking up block hash by sequence number` - Check parent chain endpoint

For detailed troubleshooting information, please refer to the individual node setup guides.

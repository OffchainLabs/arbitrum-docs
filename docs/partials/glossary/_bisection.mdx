A move in the [BoLD](/intro/glossary#bold) [Challenge Protocol](/intro/glossary#challenge-protocol) in which an [edge](/intro/glossary#edge) is split in half, producing two child edges with shorter histories. Repeated bisection narrows a disagreement down to a single execution step, which can then be resolved with a [One-Step Proof](/intro/glossary#one-step-proof). BoLD's bisection generalises the older Arbitrum Classic [dissection](/intro/glossary#dissection) move.

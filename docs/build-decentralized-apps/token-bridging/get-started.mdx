Token bridging is a fundamental aspect of any child chain protocol. It allows projects to quickly integrate with the Arbitrum ecosystem by leveraging their existing parent chain tokens.

<VanillaAdmonition type="info">

**Audience and scope.** These guides describe how to make a token bridgeable on Arbitrum One. The Solidity interfaces, SDK calls, and on-chain mechanics generalize to any Arbitrum chain — including Arbitrum Nova and chains you operate yourself — but the privileged registration path for non-upgradeable parent chain tokens differs:

- **Arbitrum One or Nova:** registration goes through Arbitrum DAO governance.
- **An Arbitrum chain you operate:** as the chain owner, you can register tokens directly through your chain's `UpgradeExecutor`. No DAO vote required.

The gateway you choose below is the same in either case.

</VanillaAdmonition>

## I want to deposit or withdraw tokens

If your token is already configured for bridging, use these guides:

- **[Deposit tokens](/build-decentralized-apps/token-bridging/deposit-tokens.mdx)**: Move tokens from the parent chain to the child chain
- **[Withdraw tokens](/build-decentralized-apps/token-bridging/withdraw-tokens.mdx)**: Move tokens from the child chain back to the parent chain

These guides work for any token that's already bridgeable, regardless of which gateway type was configured.

## I want to make my token bridgeable

If you're setting up bridging for a new token, choose a gateway based on what your token needs:

| Your token                                                                                                | Gateway                                                                                                              | Why                                              |
| --------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| Standard **ERC-20**, no special transfer behavior                                                         | [Standard](/build-decentralized-apps/token-bridging/configure-token-bridging/setup-standard-gateway.mdx)             | Automatic, no pre-configuration                  |
| Rebasing, fee-on-transfer, transfer hooks, or non-standard mint/burn                                      | [Generic-custom](/build-decentralized-apps/token-bridging/configure-token-bridging/setup-generic-custom-gateway.mdx) | Standard gateway can't represent these correctly |
| Standard transfer behavior but you need a custom child chain token (extra methods, role-gated mint, etc.) | [Generic-custom](/build-decentralized-apps/token-bridging/configure-token-bridging/setup-generic-custom-gateway.mdx) | You control the child chain token contract       |
| Custom deposit/withdraw mechanics, child-chain-native tokens, or other bridge-level behavior              | [Custom](/build-decentralized-apps/token-bridging/configure-token-bridging/setup-custom-gateway.mdx)                 | You implement the gateway itself                 |

Each option is described in more detail below.

### 1. Standard gateway (recommended for most tokens)

Use the [standard gateway](/build-decentralized-apps/token-bridging/configure-token-bridging/setup-standard-gateway.mdx) if you want:

- Automatic deployment of a standard **ERC-20** token on Arbitrum
- No pre-configuration required
- The simplest setup process

For additional information, see the [standard **ERC-20** gateway](/how-arbitrum-works/deep-dives/token-bridging.mdx#default-standard-bridging) in the conceptual docs.

### 2. Generic-custom gateway (for custom token logic)

Use the [generic-custom gateway](/build-decentralized-apps/token-bridging/configure-token-bridging/setup-generic-custom-gateway.mdx) if you need:

- Custom functionality in your child chain token
- Control over the child chain token implementation
- Additional features beyond standard **ERC-20**

For additional information, see the [Arbitrum generic custom gateway](/how-arbitrum-works/deep-dives/token-bridging.mdx#the-arbitrum-generic-custom-gateway) in the conceptual docs.

### 3. Custom gateway (for advanced use cases)

Use a [custom gateway](/build-decentralized-apps/token-bridging/configure-token-bridging/setup-custom-gateway.mdx) if you need:

- Specialized gateway logic beyond what standard gateways provide
- Custom handling of deposits and withdrawals
- Advanced features like child chain-minted tokens being withdrawable to the parent chain

For additional information, see the [other gateway types](/how-arbitrum-works/deep-dives/token-bridging.mdx#other-flavors-of-gateways) in the conceptual docs.

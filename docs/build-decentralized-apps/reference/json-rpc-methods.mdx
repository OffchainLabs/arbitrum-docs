This page lists of the JSON-RPC namespaces and methods exposed by Arbitrum Nitro nodes. Which namespaces actually accept connections depends on the `--http.api`, `--ws.api`, `--auth.api` flags passed to the node.

## Standard Ethereum namespaces

### `eth` namespace

Defined primarily in `go-ethereum/internal/ethapi/api.go`, with Arbitrum extensions in `go-ethereum/arbitrum/`.

#### Chain/block reads (`BlockChainAPI`)

| Method                                             | Description                                                                                                                      |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `eth_chainId`                                      | Returns the L2 chain ID.                                                                                                         |
| `eth_blockNumber`                                  | Latest block number.                                                                                                             |
| `eth_getBlockByNumber(blockNum, fullTx)`           | Block body. See [Blocks](/build-decentralized-apps/arbitrum-vs-ethereum/03-rpc-methods.mdx#blocks) for Arbitrum-specific fields. |
| `eth_getBlockByHash(hash, fullTx)`                 | Same as above by hash.                                                                                                           |
| `eth_getHeaderByNumber` / `eth_getHeaderByHash`    | Block headers.                                                                                                                   |
| `eth_getBlockReceipts(blockNumOrHash)`             | All receipts for a block.                                                                                                        |
| `eth_getBlockTransactionCountByNumber` / `…ByHash` | Transasction count per block.                                                                                                    |
| `eth_getUncleByBlockNumberAndIndex` / `…ByHash…`   | Always empty (no uncles on Arbitrum).                                                                                            |
| `eth_getUncleCountByBlockNumber` / `…ByHash`       | Always `0`.                                                                                                                      |
| `eth_config`                                       | Chain configuration parameters.                                                                                                  |

See [Blocks](/build-decentralized-apps/arbitrum-vs-ethereum/03-rpc-methods.mdx#blocks) for the Arbitrum-specific fields added to block responses.

#### Account/state reads

| Method                                              | Description                           |
| --------------------------------------------------- | ------------------------------------- |
| `eth_getBalance(addr, blockNumOrHash)`              | Account balance.                      |
| `eth_getCode(addr, blockNumOrHash)`                 | Contract code.                        |
| `eth_getStorageAt(addr, key, blockNumOrHash)`       | Storage slot.                         |
| `eth_getProof(addr, storageKeys[], blockNumOrHash)` | Merkle proof for account and storage. |
| `eth_getTransactionCount(addr, blockNumOrHash)`     | Account nonce.                        |

#### Execution/simulation

| Method                                                                     | Description                                                                   |
| -------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| `eth_call(args, blockNumOrHash?, stateOverrides?, blockOverrides?)`        | Offchain call.                                                                |
| `eth_estimateGas(args, blockNumOrHash?, stateOverrides?, blockOverrides?)` | Estimate gas (includes L1 calldata cost on Arbitrum).                         |
| `eth_simulateV1(opts, blockNumOrHash?)`                                    | Multi-block simulation ([EIP-7766](https://eips.ethereum.org/EIPS/eip-7766)). |
| `eth_createAccessList(args, blockNumOrHash?, stateOverrides?)`             | Generate [EIP-2930](https://eips.ethereum.org/EIPS/eip-2930) access list.     |

#### Transactions (`TransactionAPI`)

| Method                                                                            | Description                                                                                                                                                                      |
| --------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `eth_getTransactionByHash(hash)`                                                  | Transaction by hash. See [Transactions](/build-decentralized-apps/arbitrum-vs-ethereum/03-rpc-methods.mdx#transactions) for Arbitrum-specific transaction types and fields.      |
| `eth_getTransactionByBlockNumberAndIndex` / `…ByBlockHashAndIndex`                | Transaction by position.                                                                                                                                                         |
| `eth_getRawTransactionByHash` / `…ByBlockNumberAndIndex` / `…ByBlockHashAndIndex` | Raw Transaction bytes.                                                                                                                                                           |
| `eth_getTransactionReceipt(hash)`                                                 | Receipt body. See [Transaction receipts](/build-decentralized-apps/arbitrum-vs-ethereum/03-rpc-methods.mdx#transaction-receipts) for Arbitrum-specific fields.                   |
| `eth_sendTransaction(args)`                                                       | Requires unlocked local account (rare on Nitro).                                                                                                                                 |
| `eth_sendRawTransaction(rawTx)`                                                   | Submit signed transaction.                                                                                                                                                       |
| `eth_sendRawTransactionSync(rawTx, timeoutMs?)`                                   | **Nitro extension** — submit and wait for the receipt synchronously.                                                                                                             |
| `eth_sendRawTransactionConditional(rawTx, options)`                               | **Arbitrum extension** (`go-ethereum/arbitrum/conditionaltx.go`). Accepts `knownAccounts`, `blockNumberMin/Max`, `timestampMin/Max` preconditions; rejects if state has drifted. |
| `eth_fillTransaction` / `eth_signTransaction` / `eth_sign`                        | Local signing operations.                                                                                                                                                        |
| `eth_pendingTransactions()`                                                       | List pending transactions.                                                                                                                                                       |
| `eth_resend(args, gasPrice?, gasLimit?)`                                          | Replace a pending Transaction.                                                                                                                                                   |

#### Fee/gas oracle (`EthereumAPI`)

| Method                                                     | Description                                                                                                                                   |
| ---------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `eth_gasPrice`                                             | Suggested gas price (derived from L2 pricing model in Nitro).                                                                                 |
| `eth_maxPriorityFeePerGas`                                 | Always `0` on Arbitrum (no priority fee market).                                                                                              |
| `eth_feeHistory(blockCount, lastBlock, rewardPercentiles)` | Historical fee data.                                                                                                                          |
| `eth_blobBaseFee`                                          | Current blob base fee.                                                                                                                        |
| `eth_syncing`                                              | Sync status. See [`eth_syncing`](/build-decentralized-apps/arbitrum-vs-ethereum/03-rpc-methods.mdx#eth_syncing) for Arbitrum-specific fields. |

#### Filters and subscriptions (`FilterAPI`, `go-ethereum/eth/filters/api.go`)

Poll-based:

| Method                                     | Description                      |
| ------------------------------------------ | -------------------------------- |
| `eth_newFilter(crit)`                      | Create a log filter.             |
| `eth_newBlockFilter()`                     | Filter for new blocks.           |
| `eth_newPendingTransactionFilter(fullTx?)` | Filter for pending transactions. |
| `eth_getFilterChanges(id)`                 | Poll changes since last poll.    |
| `eth_getFilterLogs(id)`                    | Get all logs matching filter.    |
| `eth_getLogs(crit)`                        | One-shot log query.              |
| `eth_uninstallFilter(id)`                  | Remove a filter.                 |

Websocket subscriptions (`eth_subscribe("<topic>", …)`):

| Topic                    | Description                                                   |
| ------------------------ | ------------------------------------------------------------- |
| `newHeads`               | New block headers.                                            |
| `logs`                   | New logs matching filter.                                     |
| `newPendingTransactions` | New pending tx hashes (or full transaction if `fullTx=true`). |
| `transactionReceipts`    | **Arbitrum extension**—streams receipts as they're produced.  |

#### Accounts (`EthereumAccountAPI`)

| Method         | Description                          |
| -------------- | ------------------------------------ |
| `eth_accounts` | Usually empty on a public Nitro RPC. |

### `net` namespace

In `go-ethereum/arbitrum/net_api.go` (Arbitrum override).

| Method          | Description                                                                  |
| --------------- | ---------------------------------------------------------------------------- |
| `net_version`   | Returns the chain ID as a string (Arbitrum override—no devp2p "network ID"). |
| `net_listening` | Always `true`.                                                               |
| `net_peerCount` | Always `0` (Arbitrum nodes don't peer over devp2p).                          |

### `txpool` namespace

In `go-ethereum/arbitrum/txpool_api.go`. Nitro doesn't have a real txpool (it has a sequencer publisher and a forwarder), so these return empty structures, kept for tooling compatibility.

| Method                     | Description                             |
| -------------------------- | --------------------------------------- |
| `txpool_content()`         | Pending/queued contents (always empty). |
| `txpool_contentFrom(addr)` | Per-address contents (always empty).    |
| `txpool_status()`          | Counts (always zero).                   |
| `txpool_inspect()`         | Human-readable summary (always empty).  |

### `debug` namespace

Two sources contribute methods to this namespace.

#### From `go-ethereum/internal/ethapi/api.go` (`DebugAPI`)

| Method                                 | Description                          |
| -------------------------------------- | ------------------------------------ |
| `debug_getRawHeader(blockNumOrHash)`   | Raw RLP-encoded header.              |
| `debug_getRawBlock(blockNumOrHash)`    | Raw RLP-encoded block.               |
| `debug_getRawReceipts(blockNumOrHash)` | Raw RLP-encoded receipts.            |
| `debug_getRawTransaction(hash)`        | Raw RLP-encoded transaction.         |
| `debug_printBlock(num)`                | Human-readable block dump.           |
| `debug_chaindbProperty()`              | Chain DB property string.            |
| `debug_chaindbCompact()`               | Compact the chain DB.                |
| `debug_setHead(num)`                   | Rewind head (dangerous; admin-only). |

#### From `go-ethereum/eth/tracers/api.go` (`API`)

| Method                                                                 | Description                                                                                                                                                                              |
| ---------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `debug_traceTransaction(hash, config?)`                                | Trace one tx by hash. See [`debug_traceTransaction`](/build-decentralized-apps/arbitrum-vs-ethereum/03-rpc-methods.mdx#debug_tracetransaction) for the Arbitrum-specific `stylusTracer`. |
| `debug_traceCall(args, blockNumOrHash, config?)`                       | Trace an offchain call.                                                                                                                                                                  |
| `debug_traceBlock(rlpBytes, config?)`                                  | Trace a block from RLP.                                                                                                                                                                  |
| `debug_traceBlockByNumber(num, config?)`                               | Trace block by number.                                                                                                                                                                   |
| `debug_traceBlockByHash(hash, config?)`                                | Trace block by hash.                                                                                                                                                                     |
| `debug_traceBlockFromFile(path, config?)`                              | Trace block read from a file.                                                                                                                                                            |
| `debug_traceChain(start, end, config?)`                                | Subscription—trace a chain segment.                                                                                                                                                      |
| `debug_traceBadBlock(hash, config?)`                                   | Trace a known-bad block.                                                                                                                                                                 |
| `debug_standardTraceBlockToFile` / `debug_standardTraceBadBlockToFile` | Dump standard trace to disk.                                                                                                                                                             |
| `debug_intermediateRoots(hash, config?)`                               | State roots between txs.                                                                                                                                                                 |

Tracer config supports `callTracer`, `prestateTracer`, `flatCallTracer`, and custom JS tracers. Arbitrum's Stylus calls and ArbOS internal calls appear in traces.

### `web3` namespace

Registered by the geth node stack itself.

| Method               | Description            |
| -------------------- | ---------------------- |
| `web3_clientVersion` | Client version string. |
| `web3_sha3(data)`    | Keccak-256 hash.       |

## Arbitrum-specific namespaces

### `arbtrace` Namespace

A Parity-trace-API-compatible forwarder (`ArbTraceForwarderAPI`, `execution/gethexec/api.go:361`). Forwards calls to a "classic" (pre-Nitro) node for blocks that pre-date the Nitro upgrade, since those blocks were produced by a different VM. Configured via `RPC.ClassicRedirect` (URL of the classic node) and `RPC.ClassicRedirectTimeout`.

All methods forward verbatim and reject any request for a post-Nitro block.

| Method                                                         | Description                         |
| -------------------------------------------------------------- | ----------------------------------- |
| `arbtrace_call(callArgs, traceTypes, blockNumOrHash)`          | Trace an offchain call.             |
| `arbtrace_callMany(calls, blockNumOrHash)`                     | Trace multiple calls.               |
| `arbtrace_replayBlockTransactions(blockNumOrHash, traceTypes)` | Replay all transactions in a block. |
| `arbtrace_replayTransaction(txHash, traceTypes)`               | Replay one transaction.             |
| `arbtrace_transaction(txHash)`                                 | Get traces for a transaction.       |
| `arbtrace_get(txHash, path)`                                   | Get a specific trace at a path.     |
| `arbtrace_block(blockNumOrHash)`                               | All traces in a block.              |
| `arbtrace_filter(filter)`                                      | Filter-based trace query.           |

For post-Nitro blocks, use `debug_trace*` with the `flatCallTracer` to get the equivalent Parity-style trace.

## Notes

1. **Split architecture.** The "split" arch — execution node and consensus node — means some `arb` methods only exist on one half. On a monolithic node both are present; if you're hitting the execution side of a split deployment, you won't see `arb_getL1Confirmations` or `arb_findBatchContainingBlock`.

2. **Arbitrum-specific additions to `eth`.** `eth_sendRawTransactionConditional`, `eth_sendRawTransactionSync`, and `eth_subscribe("transactionReceipts", …)` are the most notable Arbitrum-specific additions to the standard `eth` namespace, beyond the `l1BlockNumber` and `gasUsedForL1` fields stitched into block/receipt responses.

3. **Tracing pre-Nitro blocks.** For blocks before the Nitro upgrade, use the `arbtrace_*` methods (which forward to a classic node). For post-Nitro blocks, use `debug_trace*` with `flatCallTracer` for Parity-style output.
